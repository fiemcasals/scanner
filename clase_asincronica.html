El Problema y la Meta

Entregar una PC a un operador para controlar ingresos físicos implica severos riesgos de seguridad y posibles distracciones en el trabajo.

Objetivo de la clase: Construir una terminal dedicada que solo corra una aplicación, que no use bases de datos costosas y que esté alojada gratis en internet.

Módulo 1: SO Físico

Para atrapar al usuario, empleamos el Modo Kiosco (Kiosk Mode).

Esta función nativa convierte una PC estándar en una terminal de propósito único, bloqueando por completo la interfaz gráfica del Sistema Operativo.

(Presiona la flecha ABAJO para ver los pasos de Windows)

Pasos en Windows 10/11

  1. Abre la consola CMD como Administrador.
  2. Crea rápidamente un usuario local restringido:
    net user Operador Contraseña /add
  3. Ve a Configuración > Cuentas > Familia y otros usuarios > Configurar un quiosco.
  4. Elige al usuario "Operador" y asigna el navegador Edge en modo "Señalización digital interactiva" con la URL de tu app.

Resultado: El usuario queda atrapado en pantalla completa sin barra de tareas y no puede minimizar nada.

Webhook con Apps Script

Este código .gs intercepta los datos que llegan de la cámara.


function doPost(e) {
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  try {
    var qrContent = e.parameter.codigo;

    if (sheet.getLastRow() === 0) {
      sheet.appendRow(["Fecha y Hora", "Contenido del QR"]);
    }

    sheet.appendRow([new Date(), qrContent]);
    
    return ContentService.createTextOutput("Éxito");
  } catch (error) {

  }
}

Módulo 3: Aplicación Frontend

La interfaz del operador es una SPA (Single Page App) en HTML5/JS que usa la librería html5-qrcode para acceder a la cámara web y escanear entradas físicas.

La Mecánica de Envío

Este código debe ir dentro del archivo index.html, abajo de todo, encerrado en etiquetas <script>.

Al escanear un QR, esta función manda los datos a Google Sheets usando fetch().


function onScanSuccess(decodedText) {
    const formData = new URLSearchParams();
    formData.append('codigo', decodedText);

    fetch("TU_URL_DE_APPS_SCRIPT", {
        method: 'POST',
        mode: 'no-cors', // Fundamental: Evita el bloqueo de origen cruzado
        body: formData,
        headers: { 'Content-Type': 'application/x-www-form-urlencoded' }
    }).then(() => {
        alert("¡Registro guardado con éxito!");
    });
}

Módulo 4: Hosting con GitHub Pages

Nuestra interfaz HTML necesita residir en internet para funcionar. GitHub ofrece alojamiento estático de alta disponibilidad 100% gratuito.

  1. Sube tu código (index.html) a un repositorio público.
  2. En tu repositorio, ve a la pestaña Settings > Pages.
  3. En "Source", selecciona la rama main y dale a Guardar.

En 2 minutos, la web estará pública a nivel mundial.

Módulo 5: Git y Trabajo Colaborativo

Para manipular todo esto como un desarrollador profesional, requerimos el motor de control de versiones Git en nuestra máquina.

Instalación de Git

En Linux (Ubuntu/Debian):

sudo apt update && sudo apt install git -y

En Windows (CMD / PowerShell):

winget install --id Git.Git -e --source winget

Clonar el repositorio

Para descargar exactamente este proyecto a tu PC:


git clone https://github.com/fiemcasals/scanner.git

Esto creará una carpeta idéntica con el código fuente y las guías.

Pushear (Subir tus cambios)

Si alteraste el código localmente, este es el flujo para publicarlo:


git add .
git commit -m "Explicación del cambio (ej: Agrego color rojo)"
git push origin main

Aviso de Seguridad: GitHub deprecó las contraseñas clásicas. Al hacer "push", en el campo de contraseña debes pegar tu Personal Access Token (PAT).

Crear Token (PAT)

Las contraseñas de tu cuenta ya no sirven en Git por seguridad. Debes usar un Personal Access Token:

  1. Ve a GitHub: Settings > Developer Settings.
  2. Entra a Personal access tokens > Tokens (classic).
  3. Haz clic en Generate new token (classic).
  4. Marca la casilla `repo` (para dar permisos de escritura y lectura).
  5. Haz clic en Generate y copia el código que empieza con ghp_.

Al hacer `git push`, pega ese código cuando la consola te pida tu "Password".
